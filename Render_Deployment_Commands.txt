# Esthington CRM: Render Production Deployment Commands

These are the commands you need to run on your Render Production Shell to safely update the database schema and migrate your existing data to be compatible with the newest features (ID Cards, Multi-Tier Commissions, etc.).

Please run them exactly in this order:

### 1. Update Database Schema
(This pushes the new fields like 'referrer', 'salesReferred', and company abbreviations to the live database)
npx prisma db push

### 2. Seed the Corporate ID Card Templates
(This saves the new Royalton and NEFT Properties ID HTML templates into the database so the Super Admin can select them)
npx tsx seed_neft_templates.ts

### 3. Migrate Sales Ownership Data
(This fixes old sales data to ensure they correctly show up in the new Accountant Commission reports, since we recently decoupled marketerId from leadId)
npx tsx migrate_sales_ownership.ts

### 4. Generate Employee IDs
(This assigns the correct staff IDs with company abbreviations to all existing users in the production database)
npx tsx scripts/assignEmployeeIds.ts

---
Note: None of these scripts will delete or ruin your existing data; they simply fill in the blanks where the older data is missing the new fields we created.
